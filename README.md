# Metagenome Analysis Pipeline for Long-Read Sequencing

A **Nextflow-based pipeline** for analysing **long-read bacterial metagenomic data**, including quality control, host read removal, taxonomic profiling, assembly, binning, MAG quality assessment, taxonomic classification, and antimicrobial resistance (AMR) gene detection.

## Overview

This pipeline is designed for **long-read metagenomic sequencing data** (e.g. Oxford Nanopore) and supports:

* Taxonomic profiling
* Metagenome assembly
* Metagenome-assembled genome (MAG) recovery
* MAG quality assessment
* Taxonomic classification of MAGs
* Detection of antimicrobial resistance genes

The workflow is fully reproducible and scalable using **Nextflow** and **conda/mamba environments**.

## Workflow Summary

![Workflow](/home/james/repos/github/metagenome/workflow.svg)

1. **QC & Preprocessing**: `fastplong`
2. **Host read removal**:`minimap2`
3. **Taxonomic classification and abundance estimation**: `Kraken2` and `Bracken`
4. **Visualization**: `Krona`
5. **Metagenomic binning**: `VAMB`
6. **Assembly**: *De novo* `Flye`
7. **MAG quality assessment**: `CheckM2`
8. **Taxonomic classification of MAGs**: `GTDB-Tk`
9. **AMR Detection**: `ABRicate`

## Tools Used

The pipeline integrates the following tools:

* **fastplong** - Long-read quality control and filtering
* **kraken2** - Taxonomic classification
* **bracken** - Abundance estimation refinement
* **krona** - Interactive visualisation of taxonomic profiles
* **flye** - Long-read metagenome assembly
* **vamb** - Metagenomic binning
* **checkm2** - MAG quality assessment
* **minimap2** - Host read removal and indexing
* **gtdb-tk** - Taxonomic classification of MAGs
* **abricate** - Antimicrobial resistance gene detection

## Installation

### Clone the Repository

```bash
git clone https://github.com/Aduboahen/bacteria_metagenomics.git
cd bacteria_metagenomics
```

### Create Conda/Mamba Environments

Ensure `mamba` is installed, then create the required environments:

```bash
mamba create -n bacteria_meta -f main.yml
mamba create -n bacteria_meta -f assembly.yml
mamba create -n bacteria_meta -f checkm2.yml
```

## Directory Structure

Recommended project structure:

```text

├── input/          # Raw FASTQ files
├── databases/      # Kraken2, CheckM2, and GTDB databases
├── host/           # Host genome FASTA and minimap2 index
├── results/        # Pipeline outputs
├── modules/
├── main.nf
├── nextflow.config
└── README.md
```

---

## Sample Preparation

1. **Input reads**

   * Place long-read FASTQ files in the `./input` directory.
2. **Databases**

   * Download and place required databases in `./databases`.
3. **Host genome (optional but recommended)**

   * Place the host genome FASTA file in `./host`.
   * Create a minimap2 index:

     ```bash
     minimap2 -d <host.mmi> <host.fasta.gz>
     ```

---

## Running the Pipeline

Before running, review and update paths in `nextflow.config`, including:

* Input directory
* Output directory
* Database locations
* Host genome index

Execute the pipeline with:

```bash
nextflow run main.nf
```

---

## Outputs

Key outputs generated by the pipeline include:

* Quality-controlled reads
* Taxonomic abundance tables and Krona plots
* Assembled contigs
* MAG bins
* MAG quality reports (completeness & contamination)
* GTDB-based taxonomic assignments
* AMR gene presence/absence reports

All results are organised within the `results/` directory.
